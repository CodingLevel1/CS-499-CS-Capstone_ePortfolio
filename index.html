CS-410 Software Reverse Engineering

Category: Software Design and Engineering

This project focuses on the implementation and optimization of key data structures and algorithms, particularly the Binary Search Tree (BST) and AVL Tree, to manage a collection of course data for a course planner system. The goal of the project was to efficiently store, retrieve, and manage course data while ensuring optimal performance in search and insertion operations, even with large datasets. The initial version of the project used a standard Binary Search Tree (BST) but faced limitations in performance as the dataset grew. To address these limitations, I enhanced the program by implementing an AVL Tree, a self-balancing binary search tree, to improve the efficiency of insertion and search operations. This enhancement reduced the time complexity of both operations from O(n) in the basic BST to O(log n) in the AVL Tree, which ensures better scalability when handling large amounts of course data. This project highlights my ability to apply algorithmic principles and data structure optimization in real-world scenarios. It also showcases my skills in addressing performance issues in software applications and improving the overall user experience through better scalability.

For the Software Design and Engineering category, I selected my artifact from the CS-410 Software Reverse Engineering course, which was created in February 2025. This project involved implementing security features, including user access control, input validation, and unauthorized access attempt logging. The original project was functional but lacked robust security practices, which made it a great candidate for improvement.

These enhancements not only improve the security of the software but also demonstrate my ability to adapt and implement security features in real-world software development.

Reflection:

Through this enhancement, I learned the significance of incorporating robust security practices in software design. Additionally, implementing a logging mechanism to track unauthorized access attempts proved challenging as it required me to balance the need for detailed logs with the risk of exposing sensitive information. The process was a valuable learning experience and gave me hands-on practice with security principles, which will be essential in my career.

Enhancement Two: Algorithms and Data Structures

Artifact Description:

For this enhancement, I selected my artifact from the CS-300 Data Structures course, created in August 2024. The project involved using a Binary Search Tree (BST) to manage course data for a course planner program. The program allowed users to load courses from a CSV file, store them in a tree structure, and print the course list in alphanumeric order. Initially, the program worked well for smaller datasets, but as the dataset grew, I identified a need for improvement to handle larger datasets more efficiently.

Inclusion Justification:

I included this artifact in my ePortfolio because it demonstrates my understanding of data structures and algorithms, specifically with the implementation of a Binary Search Tree. As the dataset grew, I realized that the BST could lead to inefficient performance, which led me to enhance the program by switching to an AVL tree, a self-balancing binary search tree. This change improved the program’s efficiency, particularly in insertion and search operations, by ensuring that the tree remained balanced even as new courses were added.

Course Outcomes:

The improvements made to this artifact align with Outcome 3 (Design and evaluate computing solutions), particularly in applying algorithmic principles to solve performance problems in data structures. By transitioning to an AVL tree, I optimized the program's search and insertion operations, demonstrating my ability to manage trade-offs in data structure design. Additionally, this project contributed to Outcome 4 (Use innovative techniques and tools), as implementing the AVL tree represents an innovative solution to improving the scalability and efficiency of the program.

Reflection:

This enhancement allowed me to deepen my understanding of self-balancing trees and their significance in optimizing performance. One challenge I encountered was ensuring that the AVL tree remained balanced after every insertion, which involved implementing rotations. I had to carefully follow the AVL tree’s balance factor rules and apply the correct rotations. I also faced challenges in ensuring that the program could handle invalid inputs when reading course data. Overall, this project was a great opportunity to improve my problem-solving and algorithm optimization skills.
